hexo+github搭建个人博客

发表于 2018-04-05 | 评论数：

关于搭建个人博客，好久之前就有这种想法，最近终于决定要搭建自己的博客.在网上各种查找，终于找到了一个快速开发静态博客的框架。

hexo

作为一个互联网从业者，相信好多人也和我一样，为了搭建自己的博客而苦恼；原因有很多，时间、精力、资源，相信没有一个人希望花费太多的时间和精力去维护管理。

经多方查询及朋友的经验，尝试着使用hexo框架来搭建我的个人博客(关于hexo).从hexo.io 首页，我们可以看出，hexo是一个基于 nodejs、markdown 且具有丰富插件的博客框架；so，它也是可以部署到服务器上运行的（后面会详细阐述）。

安装nodejs

  • 到nodejs网站下载nodejs运行环境，根据自己本地系统选择下载安装那个版本；

    打开终端输入 node -v 查看nodejs是否安装成功；如果展示版本号说明安装成功。

    安装nodejs时被自动安装npm,由于npm在国内访问速度问题,此时我们要将npm映射到国内的cnpm（淘宝的国内npm库）。
    eg:终端输入npm install -g cnpm --registry=https://registry.npm.taobao.org

    安装hexo

    终于要安装我们的hexo了，有没有很激动。
    首先通过终端使用命令cnpm install hexo-cli -g来安装我们hexo的命令行界面；
    等待安装完毕，是不是很简单？

  • 创建我们的hexo项目；

    通过hexo init来实现，hexo init命令是初始化hexo项目的命令，通过该命令你能在本地指定文件夹中创在项目；so,在执行明命令前你要通过cd命令来切换到你想创建项目的文件夹，也可以使用hexo init myblog在文件夹下创建myblog的hexo命令。

    使用cd myblog命令进入项目中，然后在通过cnpm install命令安装必要的nodejs组件。

    hexo server命令来启动我们的hexo服务吧！

    hexo-admin

    hexo-amdin是hexo的比较重要的插件，通过它你可以在线编辑你的博文并发布；

  • 安装hexo-admin

    cnpm install --save hexo-admin 安装hexo-admin,hexo s -d启动项目，在浏览器中打开路径“localhost:4000/admin”就能进入你的后台管理；hexo-admin支持posts、pages 及deploy，posts用来编辑你的博文，pages用来编辑添加你的页面，deploy是作为远程发布使用；

    posts、pages是用来编辑你的博文内容并实现发布；new posts/new pages 后，进入编辑页面，我们会发现，左侧是一个markdown的文本编辑器，
    右侧会动态的展示编译后的内容，是不是非常的方便；同时你会发现在右上角有一个Unpublish/publish按钮，点击之后你就后发现，在你的博客（localhost:4000）中已经出现了你刚刚编辑的博文了。

deploy，在使用该功能之前，你需要添加hexo-deployer-git插件，cnpm install hexo-deployer-git --save；可以用来执行你在服务器上定义好的ssh命令；（此意在将博客动态发布到github上），此前你需要在服务器本地做相应的配置：

  1. 在项目根目录下添加 hexo-deploy 文件，内容如下：

    hexo clean && hexo generate && hexo deploy

    hexo clean命令是用来清理服务器本地public文件夹下的生成的静态内容的， hexo generate用来生成你的博客静态内容，hexo deploy是用来执行发布操作；
    2.在配置文件_config.yml中添加配置文件如下：

    admin:
 deployCommand: './hexo-deploy'

    并在配置文件中添加你的github仓库配置，

    deploy:
  type: git
  repo: github ssh

    将本地服务器的git公共秘钥添加到你的github仓库中；

    此时在deploy中点击 deploy按钮 等待发布完成；完成之后你会发现你的静态博文已经更新到github上了。
